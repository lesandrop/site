GitHub Classroom será descontinuado: impactos para educadores e pesquisadores

Por Lesandro Ponciano , em junho de 2026.

O GitHub anunciou que fará a transição da ferramenta "GitHub Classroom" para parceiros.  O anúncio diz (https://github.com/orgs/community/discussions/196615):"Para muitos educadores que usam o GitHub Classroom, vocês devem saber que este produto está em "modo de manutenção" há 18 meses. Sabemos que isso tem sido frustrante e queremos garantir que o Classroom receba o investimento que merece. Por isso, estamos anunciando um plano de transição do GitHub Classroom para soluções de parceiros que possam atender às suas necessidades de forma mais confiável." (original em inglês: "For many educators using GitHub Classroom, you may know that this product has been operating under ‘maintenance mode’ for the past 18 months. We know this has been frustrating, and we want to make sure Classroom gets the investment it deserves. That’s why we are announcing a transition plan from GitHub Classroom to partner solutions that can more reliably fulfill your needs.") Na prática, isso significa a descontinuidade da ferramenta. 

Mas o que é o GitHub Classroom? Primeiro vamos entender o contexto. O GitHub há muito tempo deixou de ser apenas um repositório de código para se transformar em um verdadeiro ecossistema multifacetado de desenvolvimento e colaboração. Hoje, ele funciona como uma central integrada onde diferentes ferramentas operam em sinergia para atender a públicos com necessidades totalmente distintas: enquanto o GitHub tradicional resolve o controle de versão e o compartilhamento de código, o GitHub Organizations estrutura grandes equipes e empresas com governança avançada, e o GitHub Classroom adapta essa mesma tecnologia para o ambiente acadêmico, automatizando a distribuição e correção de tarefas para professores e alunos. Turbinada ainda por recursos como o GitHub Actions (para automação) e o Copilot (IA), a plataforma se consolidou não como um utilitário isolado, mas como a espinha dorsal de todo o ciclo de vida do software, seja na indústria, na comunidade de código aberto ou na sala de aula.

Então o GitHub Classroom, que está sendo descontinuado, é uma ferramenta gratuita até então desenvolvida e mantida pelo GitHub, como parte do programa GitHub Education, que é um grande "guarda-chuva" que reúne benefícios, ferramentas gratuitas e programas de capacitação para estudantes, professores e instituições de ensino. Professores do mundo inteiro tinham acesso ao GitHub Classroom por meio do GitHub Education. Essa ferramenta foi criada para ajudar educadores que usam o GitHub a distribuir tarefas de programação, fornecer feedback automatizado e gerenciar trabalhos acadêmicos usando repositórios do GitHub. 

Eu conheci o GitHub Classroom em 2019. Um pouco depois participei de um processo para me tornar GitHub Campus Advisor, que é um professor credenciado pelo GitHub que dissemina o uso de ferramentas do GitHub no campus universitário em que atua. O processo de credenciamento era simples, tinha que fazer um curso online por meio de repositórios do GitHub para mostrar fluência no uso de ferramentas Git, como clone commit,push/pull merge, e fazer uma entrevista síncrona. Na entrevista eram feitas perguntas sobre como usar GitHub na sala de aula, quais ferramentas do GitHub eram úteis aos alunos no ensino de programação. Era vantajoso fazer parte pois o professor credenciado  tinha acesso aos vários benefícios do GitHub Education.

Incentivei o uso do GitHub no campus em que eu atuava. Uma parte disso é reportada no artigo "Experiences and Insights from Using GitHub Classroom to Support Project-based Courses" (https://doi.org/10.1109/SEENG53126.2021.00013). Criei uma infraestrutura baseada nessa ferramenta que era usada em vários departamentos da universidade, reunindo dezenas de professores e milhares de alunos. Também usei a ferramenta localmente em algumas disciplinas que lecionei. A ferramenta era muito útil, embora sempre passasse por várias instabilidades. Reclamações eram comuns na lista de discussões da comunidade (https://github.com/orgs/community/discussions/categories/github-education).

Em resposta à publicação do anúncio, têm surgido discussões sobre como serão os serviços providos pelos parceiros que assumirão a ferramenta assim como discussões sobre criação de alternativas open-source. Mas a interpretação que faço é que se está buscando um modelo de negócio lucrativo no GitHub e GitHub Classroom não cabe bem nessa busca. É importante lembrar que o GitHub era uma Starup que foi comprada pela Microsoft em 2018 (https://web.archive.org/web/20260322040025/https://news.microsoft.com/announcement/microsoft-acquires-github/). O GitHub classroom oferecia ao professor a escala para gerenciar turmas e isso levava ao consumo de grande quantidade de recursos gratuitamente como criação automática de até milhares de repositórios privados para seus estudantes e armazenar até terabytes de dados sem custo. Funcionou enquanto o GitHub estava explorando esse nicho de mercado, mas ao que tudo indica essa exploração acabou e não foi identificada oportunidades nos níveis de retorno que desejavam. Naturalmente, nessa análise levo em conta todo o contexto e não apenas o anúncio em si.

Para alguém que acompanha esse ecossistema há muito tempo, desde o antigo Google Code e dos sistemas CVS e o SVN (Subversion), acho que está havendo uma grande mudança nesse ecossistema. as mudanças que as IAs estão causando para quem trabalha com atividades relacionadas ao programação também chegaram aos sistemas cuja essência é o controle de versões, como é o caso do GitHub e suas ferramentas. Eu não me surpreenderei se vierem mudanças em outras ferramentas muito usadas gratuitamente como GitHub Organizations, GitHub Actions e o próprio GitHub Pages em que este Academizando está atualmente hospedado.

OBS.: Este texto foi originalmente publicado em junho de 2026. A situação do GitHub Classroom e das ferramentas parceiras pode ter evoluído desde então.
